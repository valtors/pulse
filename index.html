pulse
connect everything. your ai does the rest.
the problem

you get 38 notifications. 29 are ci failures. 3 are mentions you need to see. the rest is noise. you miss the 3 because the 38 drowned them. pulse fixes this.

how it works
$ pulse connect github ghp_xxxx connected github $ pulse digest URGENT: valtors/relay [PullRequest] docs: add migration guide - you were mentioned microsoft/autogen [Discussion] Pre-action authority receipt - you were mentioned NEEDS ATTENTION: AgentWrapper/agent-orchestrator [Issue] Daemon status polling - your issue has activity [Issue] Superseded review runs - your issue has activity NOISE: 29 items filtered (ci, status checks, automated noise) 38 became 4. the rest doesn't matter.
architecture

rust core

rust

the brain. filtering engine, sqlite memory, llm orchestration. serde, rusqlite, reqwest. fast, safe, no gc pauses.

go shell

go

the hands. http server, web ui, cli, connector management. stdlib only. one binary, zero dependencies.

two languages because each does what it's best at. the go binary calls the rust binary. they share sqlite and json. no grpc, no protobuf, no microservices.

commands
pulse connect github <token> # connect a service
pulse digest # get your filtered summary
pulse ask "what did i miss" # ask anything
pulse remember focus "ship v1" # store memory
pulse serve # web ui on localhost:9090
philosophy

local first. your data stays on your machine. single binary. no docker, no kubernetes, no saas. no telemetry. no phone home. no account. sqlite, stdlib, http. boring tech on purpose.
